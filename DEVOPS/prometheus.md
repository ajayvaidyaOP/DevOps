# 🚀 1. Introduction to Prometheus

## 🔹 What is Prometheus?

Prometheus is an open-source monitoring and alerting toolkit designed for collecting and storing metrics as time-series data.

It is widely used in:

- Cloud-native environments
- Kubernetes monitoring
- Infrastructure monitoring
- Application monitoring
- DevOps observability

Prometheus uses a pull-based model to scrape metrics from configured targets.

---

## 🔑 Key Features of Prometheus

| Feature | Description |
|---|---|
| Time-Series Database | Stores metrics with timestamps |
| Pull-Based Monitoring | Scrapes metrics from targets |
| PromQL | Powerful query language |
| Alerting | Generate alerts based on metrics |
| Exporters | Collect metrics from systems |
| Kubernetes Support | Native cloud-native monitoring |
| Service Discovery | Automatically detect targets |

---

## 🔹 Important Prometheus Definitions

| Term | Definition |
|---|---|
| Metric | Numerical data collected over time |
| Time Series | Sequence of metric values with timestamps |
| PromQL | Query language used in Prometheus |
| Exporter | Service exposing metrics for scraping |
| Scraping | Process of collecting metrics |
| Alertmanager | Handles alerts generated by Prometheus |
| Target | Endpoint from which Prometheus collects metrics |
| Rule | Configuration for alerts or recording |
| Federation | Collect metrics from multiple Prometheus servers |
| Retention | Duration metrics are stored |
| Label | Key-value metadata attached to metrics |

# 🛠️ 3. Prometheus Installation

## 🔹 Install Prometheus on Linux

### Download Prometheus

```bash
wget https://github.com/prometheus/prometheus/releases/download/v2.30.0/prometheus-2.30.0.linux-amd64.tar.gz
```

---

### Extract Files

```bash
tar xvfz prometheus-*.tar.gz
```

---

### Start Prometheus

```bash
cd prometheus-*
./prometheus --config.file=prometheus.yml
```

---

## 🐳 Run Prometheus Using Docker

```bash
docker run -p 9090:9090 prom/prometheus
```

---

## 🌐 Access Prometheus UI

Open browser:

```text
http://localhost:9090
```

---

# ⚙️ 4. Prometheus Configuration

## 🔹 Basic prometheus.yml Configuration

```yaml
global:
  scrape_interval: 15s

scrape_configs:

  - job_name: 'prometheus'

    static_configs:
      - targets: ['localhost:9090']
```

---

## 🔹 Add Node Exporter Target

```yaml
- job_name: 'node_exporter'

  static_configs:
    - targets: ['localhost:9100']
```

---

## 🔹 Configuration Sections

| Section | Purpose |
|---|---|
| global | Global Prometheus settings |
| scrape_configs | Define scrape targets |
| rule_files | Load alerting/recording rules |
| alerting | Configure Alertmanager |

# 🔍 5. PromQL (Prometheus Query Language)

## 🔹 Basic Queries

### Check Target Status

```promql
up
```

---

### HTTP Request Rate

```promql
rate(http_requests_total[5m])
```

---

## 🔹 Aggregation Queries

### Sum Requests

```promql
sum(rate(http_requests_total[5m]))
```

---

### Average Over Time

```promql
avg_over_time(http_requests_total[5m])
```

---

## 🔹 Recording Rules

```yaml
groups:

- name: example

  rules:

  - record: job:http_inprogress_requests:sum
    expr: sum(http_inprogress_requests) by (job)
```

---

## 🔹 Common PromQL Functions

| Function | Description |
|---|---|
| rate() | Calculate per-second increase |
| sum() | Sum values |
| avg() | Average values |
| max() | Maximum value |
| min() | Minimum value |
| histogram_quantile() | Percentile calculation |

## 🔹 Custom Exporter Example (Python)

```python
from prometheus_client import start_http_server, Gauge
import random
import time

g = Gauge('random_number', 'A random number')

def generate_random_number():

    while True:
        g.set(random.random())
        time.sleep(5)

if __name__ == '__main__':

    start_http_server(8000)
    generate_random_number()
```

# 🚨 7. Alerts and Alertmanager

## 🔹 Alerting Rules

```yaml
groups:

- name: example

  rules:

  - alert: HighMemoryUsage

    expr: node_memory_Active_bytes / node_memory_MemTotal_bytes * 100 > 90

    for: 5m

    labels:
      severity: critical

    annotations:
      summary: "High memory usage detected"

      description: "Memory usage above 90%"
```

---

## 🔹 Alertmanager Configuration

```yaml
global:
  resolve_timeout: 5m

route:
  group_by: ['alertname']
  receiver: 'email'

receivers:

- name: 'email'

  email_configs:

  - to: 'your-email@example.com'
    from: 'prometheus@example.com'
    smarthost: 'smtp.example.com:587'
```

---

## 🔹 Alertmanager Features

| Feature | Description |
|---|---|
| Alert Routing | Send alerts to teams |
| Grouping | Combine related alerts |
| Silencing | Temporarily mute alerts |
| Deduplication | Remove duplicate alerts |

# 🌐 8. Prometheus Federation

## 🔹 What is Federation?

Federation allows Prometheus servers to scrape metrics from other Prometheus servers.

Used for:

- Centralized monitoring
- Multi-cluster monitoring
- Scalability

---

## 🔹 Federation Configuration

```yaml
scrape_configs:

- job_name: 'federate'

  honor_labels: true

  metrics_path: '/federate'

  params:

    match[]:
      - '{job="prometheus"}'

  static_configs:

    - targets:
      - 'prometheus-server-1:9090'
      - 'prometheus-server-2:9090'
```

# ☸️ 9. Monitoring Kubernetes with Prometheus

## 🔹 Why Use Prometheus with Kubernetes?

Prometheus integrates deeply with Kubernetes for monitoring:

- Pods
- Nodes
- Services
- Containers
- Cluster health

---

## 🔹 Prometheus Kubernetes Deployment Example

```yaml
apiVersion: monitoring.coreos.com/v1
kind: Prometheus

metadata:
  name: prometheus

spec:

  replicas: 1

  serviceAccountName: prometheus

  serviceMonitorSelector:
    matchLabels:
      team: frontend

  resources:
    requests:
      memory: 400Mi
```

---

## 🔹 ServiceMonitor Example

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor

metadata:
  name: example-monitor

spec:

  selector:
    matchLabels:
      app: example

  endpoints:
    - port: web
```

# 🛠️ 12. Troubleshooting Prometheus

## 🔹 High Cardinality Issues

Too many unique labels can overload Prometheus.

Examples:

- Dynamic user IDs
- Session IDs

---

## 🔹 Slow Queries

Optimize queries by:

- Reducing cardinality
- Using recording rules
- Avoiding expensive aggregations

---
